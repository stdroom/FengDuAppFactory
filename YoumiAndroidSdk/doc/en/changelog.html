    Change Log¶

    1. Offerwall¶

    v5.3.3¶

    1. Fix Reward Ad can’t be Finished in some model again

    v5.3.2¶

    1. Fix Reward Ad can’t be Finished in some Model
      • Permission Configuration has been changed

    v5.3.1¶

    1. Offerwall supports Android 5.1
    2. Fix the problem that sharing task can’t be shared sometimes

    v5.3.0¶

    1. add Sharing task , share once and gain infinite future benefits

    2. From this version, SDK will no longer support following user data Statistical API, but the old versions remains this function.
      • AdManager.getInstance(Context context).setUserDataCollect(true);
    3. Client Reward Hosting system, reward type change from int -> float, it may affects following API:
      • Client Reward Management API
      • Monitor the change of Reward Balance
      • EarnPointsOrderInfo

    4. Fix other bugs

    v5.1.3¶

    1. Fix the bug when offerwall sometimes cannot displayed on some devices

    v5.1.2¶

    1. Fix the problem that SDK may cause UI crash when implementing notification on Android 5.1.0

    v5.1.0¶

    1. Offerwall supports Android 5.0
    2. Optimize display of Ad statu
    3. Increase reward arrival rate
    4. Other optimization and bug fixed

    v5.0.0¶

    1. Optimize Offerwall loading speed
    2. Support Offerwall pull-refresh and double-click-refresh
    3. Cancel initiative offerwall test mode
    4. Support custimizing part of Offerwall UI
    5. Provide full-screen offerwall exit monitoring API
    6. Add online time checking function
    7. Simplfy permission configuration
    8. Increase reward arrival rate
    9. Optimize user data static function

    v4.10¶

    1. SDK no longger support incentive banner or interstitial
    2. Simplfy Offerwall AndroidManifest.xml configuration
    3. Optimize local anti-cheating measures
    4. Optimize reward arrival
    5. Optimize ad launching mechanism
    6. Optimize Server postback implementation
    7. Solve problems that offerwall cannot display on some devices
    8. Add noification configuration switch
    9. Provide method to set new channel No.
    10. Provide new client reward order monitoring API
    11. Other optimization and bug fixed

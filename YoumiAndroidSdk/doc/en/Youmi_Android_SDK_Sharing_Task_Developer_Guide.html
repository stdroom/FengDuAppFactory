    Youmi Android SDK Sharing Task Developer Guide¶

    Tip

    Welcome to download Youmi Ad SDK. In order to ensure you receiving Youmi ad normally and guarantee you ad revenue, we may need you to stick to the steps listed in the documentation under the doc file, or please take referrance to the demo source code.

    Please read the following tips before integration:

    1. When you are creating the app on the Youmi backend to get APPID and KEY, please be noted that you MUST select the right OS;
    2. One pair of APPID and KEY only can be applied to one app; P.S: please do NOT use the same APPID and KEY in different apps, otherwise you may NOT able to receive ad normally;
    3. You can receive test ad after integration; After test and confirmation, please upload the apk/ipa on Youmi website for review; After review approval you will receive official ads that brings you revenue immediately;
    4. Please noticed that Youmi ad do NOT support Simulator, You MUST test on real mobile Device; Besides, Android pad is not appliable here for test, please test Android ad on phone only.

    1. Youmi Android SDK general configuration¶

    Before add the Sharing task, please make sure you have already finished the general configuration according to Youmi Android SDK General Configuration Developer Guide .

    2. Configure Sharing Ad Components¶

    Please copy the following configuration code to AndroidManifest.xml file:

    <activity
    android:name="net.youmi.android.AdBrowser"
    android:configChanges="keyboard|keyboardHidden|orientation|screenSize"
    android:theme="@android:style/Theme.Light.NoTitleBar" >
</activity>
<service
    android:name="net.youmi.android.AdService"
    android:exported="false" >
</service>
<service
    android:name="net.youmi.android.ExpService"
    android:exported="false" >
</service>
<receiver
    android:name="net.youmi.android.AdReceiver" >
    <intent-filter>
        <action android:name="android.intent.action.PACKAGE_ADDED" />
        <data android:scheme="package" />
    </intent-filter>
</receiver>

    3. Sharing task Code Integration (Important)¶

    Tip

    Since the release of Youmi Android Offerwall SDK V5.3.0, Youmi Offerwall supports Sharing task; Here User gets rewards when he shares the ad contents on his social networks, meanwhile, developer get relevant revenue.

    Youmi Sharing ad is paid based on CPA+CPC model; one Sharing may bring infinite rewards and revenue.

    Time of rewards and settlement:

    • （CPA）after user shared the ad content successfully, he gets the first sharing reward and developer gets the first sharing revenue;
    • （CPC）whenever new users click or share the shared link, the original user will get the rewards and developer get revenue;
    • （CPA）Part of the Sharing task supports deep-link task; whenever new users completed assigned task in the shared link, they can get rewards and developer earn revenue at the same time.

    1) Initialization of Sharing task and Resource Recycle¶

    Please start the initialization before invoke other function of Sharing Task (e.g. invoke the following code in onCreate of the first Activity)

    import net.youmi.android.offers.OffersManager;
...
OffersManager.getInstance(Context context).onAppLaunch();

    In the exit of the app (e.g. in the onDestroy method of Activity), invoke following code to complete the Sharing task Offerwall resource recycle.

    import net.youmi.android.offers.OffersManager;
...
OffersManager.getInstance(Context context).onAppExit();

    Tip

    Notice: Offerwall and Sharing Task are implementing initialization and resource recycle with the same code; As the result, if you want to integrate Offerwall and Sharing task at the same time, the relevant code only need to be invoked one time.

    2) Display the full-screen ShareWall¶

    Please invoke the following code to display the full-screen Sharewall:

    import net.youmi.android.offers.OffersManager;
...
OffersManager.getInstance(Context context).showShareWall();

// Meanwhile this method is also applied to the following reload
// Support full-screen Offerwall exit monitoring callback
OffersManager.getInstance(Context context).showShareWall(Interface_ActivityListener listener);

    Interface_ActivityListener defines as follow:：

    public interface Interface_ActivityListener {

    /**
     * full-screen sharewall Activity send callback when invoking onDestory,
     * and it will executed in the UI thread
     */
    public void onActivityDestroy(Context context);
}

    3) Display the floating half-screen Sharewall¶

    Please invoke the following code in UI thread, to display the floating half-screen Sharewall:

    import net.youmi.android.offers.OffersManager.
...
OffersManager.getInstance(Context context).showShareWallDialog(Activity activity);

// Meanwhile this method supports following reload

// input dialog to end the monitor
OffersManager.getInstance(Context context).showShareWallDialog(Activity activity, OffersWallDialogListener listener);

// input dialog Wide, height and pixel
OffersManager.getInstance(Context context).showShareWallDialog(Activity activity, int width, int height);

// input dialog wide, height and pixel, and end the monitor
OffersManager.getInstance(Context context).showShareWallDialog(Activity activity, int width, int height, OffersWallDialogListener listener);

// input the dialog wide and height proportion (0~1)
OffersManager.getInstance(Context context).showShareWallDialog(Activity activity, double scaleOfScreenWidth, double scaleOfScreenHeight);

// input the dialog wide and height proportion (0~1) and end the monitor
OffersManager.getInstance(Context context).showShareWallDialog(Activity activity, double scaleOfScreenWidth, double scaleOfScreenHeight, OffersWallDialogListener listener);

    OffersWallDialogListener define as follows:

    public interface OffersWallDialogListener {

    /**
     * send callback when end the dialog
     */
    public void onDialogClose();
}

    4. Integration the Sharing task channel (Important)¶

    Your user could shre the ad content to social network when you configure the social networks (e.g. Wechat, QQ and Weibo) with the following Guide.

    1) Configure Wechat¶

    Please follow the steps to configure Wechat share, then the ad content can be shared to Wechat platform.

    (1) import Wechat SDK¶

    unzip the SDK, and import extralibs/libammsdk.jar to the assigned libs in the project.

    (2) Configure Wechat SDK proguard setting¶

    If your project are using proguard packing, please MUST add the following code, otherwise you are not able to process the sharing normally:

    # weixin sdk
-dontwarn com.tencent.mm.sdk.**
-keep class com.tencent.mm.sdk.** {
    *;
}

    (3) Configure Wechat appid¶

    Current SDK v5.3.0 supports sharing to the Wechat Platform, developer may need to register your app on Wechat Developer platform and get the relevant APPID, and send your wechat APPID to Youmi.

    a. Apply for Wechat appid¶
    1. Please login Wechat Developer Platform and register the developer account, and apply for the Wechat appid for the current app;
    2. When you register your Android app and get the APPID on the Wechat Developer Platform, you need to provide md5 value in the official signature of your app; You can get it with the tool provided by Wechat, or you can get it by invoking this following method directly.
    import net.youmi.android.offers.OffersManager;
...

// Get the md5 in current signature, p.s.: MUST invoke this after packing with official signature file
String signatureMd5String = OffersManager.getInstance(Context context).getSignatureMd5String();
    b. Register Wechat APPID to SDK¶

    When you registered the APP and get the Wechat APPID, before implementation of the sharewall, please register Wechat APPID to Youmi by invoking with the following code (e.g. invoke the following code in onCreate of the first activity when app starts)

    import net.youmi.android.offers.OffersManager;
...

// Only returs true means successful registration, then you can implement more function normally
boolean isRegisterSuccess = OffersManager.getInstance(Context context).registerToWx(String weixinAppid);
Log.i("youmi", String.format("Register appid %s", isRegisterSuccess ? "Successed" : "Failed"));

    Caution

    There are some situations may cause returning false:

    1. Please check the Wechat proguard configuration to make sure if it has been configured correctly
    2. The current mobile phone has not been installed wechat.

    (4) Configure the Activity to recevie the sharing result¶

    When the ad content has been shared to Wechat, Wechat platform will send callback including pkgname + .wxapi.WXEntryActivity to the application which starts the sharing request; More details please refer to Wechat official instruction here

    a. edit activity code¶

    In case to notify Youmi the successful sharing result, you meed to invoke Youmi’s proceed caode in the abovementioned activity, following the super line, and rewrite onCreate(Bundle savedInstanceState) and onNewIntent(Intent intent).

    Instance coding:

    package PackageName.wxapi;

public class WXEntryActivity extends Activity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);

        // add Youmi proceed method
        OffersManager.getInstance(this).handleIntent(getIntent());

        // other proceed methods
        // ....
    }

    @Override
    protected void onNewIntent(Intent intent) {
        super.onNewIntent(intent);

        // add Youmi proceed method
        OffersManager.getInstance(this).handleIntent(getIntent());

        // other proceed methods
        // ....
    }
}
    b. Please configure activity to AndroidManifest.xml file¶
    <activity
    android:name="PackageName.wxapi.WXEntryActivity"
    android:exported="true"
    >
</activity>

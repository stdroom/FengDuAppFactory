    1. Introductions¶

    1) Environment Configuration¶

    (1) Which character coding that Youmi SDK will use¶

    • Youmi SDK will use UTF-8 character coding. Please use UTF-8 coding environment when integrating advertisement and importing sample programs. Otherwise, wrong-coding jumbles will happen.

    (2) Which Android SDK versions are compatible with Youmi SDK¶

    • Android SDK version 2.1 and above are compatible with Youmi SDK. Version below 2.1 might have some compatible issue.

    2. About Test Mode¶

    • About Incent Ad : under the test mode you will get reward returned but no revenue paid.
    • About Non-incent Ad : No limits for ad display and play, but there is no revenue for test display or play.

    The following situations shall always under test mode:

    1. Device with no exclusive IMEI (e.g. some Android pad)
    2. Application before Youmi review approval
    3. Approved by Youmi before, but change pkg name in following version but still using the previous APPID and KEY
    4. Initialize and set the app to testmode

    Tip

    P.S. After invoking initilization API, current app info will be displayed in tag:YoumiSdk of LogCat

    3. Using target-sdk 17 above to compile¶

    1. Android JavascriptInterface security holes repair¶

    Android version lower thant 4.2 will have security holes. It can make use of the special interface function addJavascriptInterface in webview to penetrate webkit to control Android machine. View addJavascriptInterface security holes statement

    Android 4.2(API 17) already use the new interface function, to repair security holes. There are official statement for JavascriptInterface in webview :

    Tip

    IMPORTANT： This method can be used to allow JavaScript to control the host application. This is a powerful feature, but also presents a security risk for applications targeted to API level JELLY_BEAN or below, because JavaScript could use reflection to access an injected object’s public fields. Use of this method in a WebView containing untrusted content could allow an attacker to manipulate the host application in unintended ways, executing Java code with the permissions of the host application. Use extreme care when using this method in a WebView which could contain untrusted content. JavaScript interacts with Java object on a private, background thread of this WebView. Care is therefore required to maintain thread safety. The Java object’s fields are not accessible.

    Caution

    Youmi Android SDK repair these security holes. Therefore, target-sdk 17 and above versions are needed to repair the security holes.

    2. Will there be any compatible issue with lower version when using target-sdk 17 and above versions to compile?¶

    • In Proguard Configuration file:
      • target works in compiling. It is used for compiling, target to which API version (SDK version) to compile.
    • in AndroidManifest.xml file：
      • minSdkVersion and maxSdkVersion work in program installation. They are used for targeting which version of devices can install this application.
      • targetSdkVersion works in program running. It is used for targeting higher version device to run program.

    Therefore, using target-sdk 17 and above versions won’t cause any compatible issue with lower version machine. Meanwhile, if you set android:targetSdkVersion and targetSdkVersion>=17 in file AndroidManifest.xml, then you need to set target-sdk to be above 17 in proguard configuration file.

    2. FAQ of Rewarded Advertisement¶

    1) Can’t open offerwall¶

    1. Confirm that offerwall configuration is correct
    2. Check proguard configuration
    3. Try to finish task with other phones, excluding the influence of specific phone

    2) Can’t acquire currency points¶

    1. Confirm that offerwall configuration is correct
    2. Check if the task is finished normally according to the task hint
    3. Check if the app is open automatically after successfully installing
    4. Try to finish task with other phones, excluding the influence of specific phone

    3) Can I test the ADs before it pass the audit¶

    Yes, you can. But the ADs are test ADs, you can get the currency points, but you can’t get money revenue for the ADs. You can get get money revenue with the formal ADs(After passing the audit).

    4) Fail in server-callback signature verification¶

    1. Check if server callback is used correctly or not
    2. Log in Youmi Developer Control Panel Confirm the appid and server-callback key is corresponding to the application.
    3. Check if using utf-8 encoding to process the received parameters
    4. Before involving in signature, confirm that the three parameters Ad Name 、 User ID 、 Deceive ID are processed by urldecode
    5. Confirm that the signature algorithm is MD5（32 bits, lower-case）

    Tip

    Be Noticed: If developers change the server call-back address url in Youmi Developer Control Panel, it will need some time for it to take effect.

    3. FAQ of Non-Incent Advertisement¶

    1) Interstitials can be loaded, but can’t be clicked and closed¶

    Please check if there is the below permission inside AndroidManifext.xml :

    <uses-permission android:name="android.permission.SYSTEM_ALERT_WINDOW" />

    4. FAQ of Sharing Task¶

    1) Can’t share to WeChat Moments¶

    Solution:

    1. The current mobile phone has no installed wechat or the version of wechat is too low to share to Moments. In addition, our sdk will show the relatived tips about this problem;
    2. If you implement Proguard configuration, please check whether the proguard file had benn successfully set Youmi SDK Proguard Setting ， Wechat SDK Proguard Configuration ;
    3. Please check whether the Signature of the current app is the same with the Signature you register in weixin website.

    2) Not rewards gained after sharing¶

    Solution:

    1. If you implement Proguard configuration, please check whether the proguard file had been successfully set Youmi SDK Proguard Setting ， Wechat SDK Proguard Configuration ；
    2. Please check whether correctly configure activity to receive the sharing result 。

    3) Return to a blank page after share¶

    Analysis:

    • After sharing the ad content to Wechat, it shall starts pkg name + .wxapi.WXEntryActivity, otherwise it may brings problem.

    Solution:

    • In onCreate(Bundle savedInstanceState) and onNewIntent(Intent intent) of pkg name + .wxapi.WXEntryActivity, you shall invoke finish() to end WXEntryActivity directly.
